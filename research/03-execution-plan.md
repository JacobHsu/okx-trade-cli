## 2. 執行架構

```
monitor.js（每 5 分鐘）
  ├─ 查詢 bot 狀態、掛單數、持倉損益
  ├─ 判斷三個觸發條件
  │    ├─ Trigger 1：價格突破區間上限 + aboveBy → 上移重建
  │    ├─ Trigger 2：價格跌破區間下限 - belowBy → 下移重建
  │    └─ Trigger 3：連續 N 次無活躍掛單 → 橫盤重建
  ├─ 通過安全閥（冷卻期 + 日交易上限）
  └─ 執行：停止舊 bot → 建立新 bot（以當前價格為中心 ±rangeHalfWidth）
```

**分工說明：**
- `monitor.js`：判斷市場狀態、決定何時重建
- `OKX bot grid`：負責掛單、補單、網格循環維護

---

## 3. 觸發條件詳述

### Trigger 1 — 上移重建

| 條件 | ETH 現價 > bot maxPx + 50 |
|------|--------------------------|
| 行為 | 停止舊 bot，以當前價格為中心重建新區間 |
| 目的 | 追蹤上漲行情，避免錯過套利機會 |

### Trigger 2 — 下移重建

| 條件 | ETH 現價 < bot minPx - 50 |
|------|--------------------------|
| 行為 | 停止舊 bot，以當前價格為中心重建新區間 |
| 目的 | 追蹤下跌行情，重新部署有效區間 |

### Trigger 3 — 橫盤重建

| 條件 | 連續 6 次檢查（30 分鐘）`liveOrders = 0` 且有成交紀錄 |
|------|------------------------------------------------------|
| 行為 | 停止舊 bot，在當前價格重建 |
| 目的 | 應對所有網格單已成交、bot 空轉的情況 |

---

## 4. 安全閥

| 規則 | 設定值 | 說明 |
|------|--------|------|
| 冷卻期 | 1 小時 | 每次重建後至少間隔 1 小時才能再次觸發 |
| 日交易上限 | 2 次 | 每日最多重建 2 次，防止頻繁交易 |
| 負價保護 | 自動 | `minPx ≤ 0` 時拒絕執行 |

---

## 5. 監控指標

| 指標 | 正常範圍 | 告警閾值 |
|------|----------|----------|
| 日往返次數 | 2–6 次 | < 1 次或 > 10 次 |
| liveOrders | ≥ 1 | = 0 持續 30 分鐘 |
| 持倉未平時間 | < 24 小時 | > 48 小時 |
| 當日損益 | -5% ~ +5% | < -5% |

## 6. 應急預案

### 場景 A：價格持續單邊上漲（突破區間上限）
- Trigger 1 自動觸發上移重建
- 若當日已達 `maxDailyTrades`，需手動介入評估

### 場景 B：價格持續下跌（突破區間下限）
- Trigger 2 自動觸發下移重建
- 下跌趨勢明確時考慮暫停策略（手動設定 `enabled: false`）

### 場景 C：橫盤無成交（掛單耗盡）
- Trigger 3 在 30 分鐘後自動觸發重建
- 若重建後仍無成交，評估市場流動性

### 場景 D：手動暫停策略
編輯 `runtime/auto-trade-config.json`，將 `enabled` 設為 `false`：
```json
{ "enabled": false }
```
