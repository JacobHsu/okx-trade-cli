# 交易策略比較：Grid Bot 與 Volatility Breakout

本專案 (`okx-trade-cli`) 目前實作了兩套完全獨立的交易策略。雖然兩者共用相同的基礎架構（如 OKX CLI、Node.js 環境），但它們的市場假設、獲利邏輯與執行週期截然不同。

以下將詳細說明這兩套策略的不同之處：

## 1. 策略基本資訊比較

| 比較維度 | 網格交易策略 (Grid Bot) | 波動率突破策略 (Volatility Breakout) |
| :--- | :--- | :--- |
| **主要腳本** | `scripts/monitor.js` | `scripts/vb-monitor.js` |
| **策略類型** | 區間震盪 (Range-bound) | 趨勢突破 (Trend Following) |
| **市場假設** | 價格會在一定區間內來回震盪 | 價格突破盤整區間後，會形成持續性趨勢 |
| **獲利來源** | 高拋低吸，賺取網格利潤 | 捕捉大波段趨勢利潤 |
| **風險暴露** | 單邊極端行情（暴漲或暴跌） | 頻繁的假突破（震盪洗盤） |
| **監控頻率** | 每 5 分鐘 | 每 30 分鐘（基於 1 小時 K 線） |
| **部署方式** | 自動監控並透過 API 重建網格 | 直接透過 API 下單、設止損/止盈 |

---

## 2. 策略邏輯解析

### 網格 Bot 策略 (Grid Bot)

網格交易是一種典型的「逆勢」微觀操作，宏觀上依賴「震盪」行情。

- **核心邏輯**：在設定好的價格區間內，將資金分成等份，每跌一定比例就買入（做多），每漲一定比例就賣出（平倉）。
- **觸發條件**：
  - 當價格突破區間上限：上移重建網格。
  - 當價格跌破區間下限：下移重建網格。
  - 當橫盤過久無活躍掛單：於當前價格附近重新建網格。
- **優勢**：在沒有明確方向的震盪市中，能持續產生穩定的現金流。
- **劣勢**：資金利用率較低（未觸發的網格資金閒置），若遇到單邊大趨勢會導致套牢或提早出場錯失利潤。

### 波動率突破策略 (Volatility Breakout, VB)

VB 策略是典型的「順勢」操作，基於技術指標過濾雜訊，捕捉大波段。

- **核心邏輯**：觀察價格波動幅度。當價格向上突破上緣時，認為多頭趨勢形成；跌破下緣時，認為空頭趨勢形成。
- **技術指標 (K 線判斷)**：
  - **Keltner Channel (肯特納通道)**：判斷價格是否突破通道上下軌。
  - **EMA (220) & ADX & RSI & Volume**：作為多重過濾條件，確認突破的有效性（避免假突破）。
- **風險控制**：採用 ATR × 4 的動態止損機制，並搭配保本移停與追蹤止損。
- **優勢**：在出現明確的牛市或熊市趨勢時，可以獲得極高的盈虧比（捕捉肥尾效應）。
- **劣勢**：勝率通常較低，在震盪盤整時期容易因為假突破而頻繁停損（被雙巴）。

---

## 3. 執行與排程差異

因為兩者的交易時間框架 (Timeframe) 不同，在系統排程上有以下差異：

* **網格 Bot**：由於需要密切監控價格是否超出網格區間，以及掛單是否過久未成交，因此排程設計為 **每 5 分鐘** 執行一次 (`setup-scheduler.ps1`)。
* **VB 策略**：技術指標的計算基於 **1 小時 (1H) K 線**。為了避免過度頻繁檢查且確認收盤價，排程設計為 **每 30 分鐘** 執行一次 (`setup-vb-scheduler.ps1`)，並在每次開倉後有 4 小時的冷卻期以避免連續進場。

## 結論

這兩個策略在資金配置上具有很好的**互補性**：
- 在震盪市中，Grid Bot 負責穩定獲利，VB 策略可能產生小額停損。
- 在單邊趨勢中，VB 策略負責賺取超額波段報酬，彌補 Grid Bot 因價格單邊突破帶來的回撤。
